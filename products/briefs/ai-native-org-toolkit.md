# 产品 Brief：AI-Native Org Toolkit

**阶段**：Idea → Validation  
**创建**：2026-06-12  
**Owner**：Baokai

---

## 🎯 问题陈述

**目标客户遇到的问题**：
- 想建立 AI 原生组织，但不知道从哪里开始
- 现有的组织框架（如 org-blueprint）过于复杂，难以上手
- 缺少可执行的工具和检查清单
- 不知道如何度量 AI 协作效果

**为什么现有方案不够好**：
- 大部分是概念框架，缺少可执行工具
- 文档丰富但实际运行指南少
- 没有度量标准，无法改进

---

## 👥 目标客户

**主要客户**：
- 一人公司创始人或 solopreneur
- 想用 AI agents 提升效率的小团队（2-5 人）
- 技术背景，熟悉 Git 和命令行

**次要客户**：
- AI 研究者（研究 AI-native organization）
- 咨询顾问（帮助客户建立 AI 协作体系）

**非目标客户**：
- 大型企业（需求太复杂）
- 非技术背景用户（需要 GUI）

---

## 💡 解决方案

**核心价值**：30 分钟搭建可运行的 AI 原生组织

**包含内容**：
1. **Org Bootstrap Kit**
   - 完整目录结构模板
   - 核心文件（NOW.md, owner.md, agent-protocol.md）
   - 自动化脚本（周报生成、Duty 巡检）
   - CI 检查配置

2. **Essential Skills Pack**（目前 2 个）
   - org-bootstrap: 组织初始化
   - duty-check: 定期巡检

3. **Quick Start Guide**
   - 10 分钟入门教程
   - 第一个任务示例
   - 能力沉淀示例

4. **Metrics Dashboard Template**（未来）
   - 沉淀率追踪
   - 任务完成率
   - Agent 工作时长

**差异化**：
- ✅ 极简（3 层结构 vs 5 层）
- ✅ 可执行（脚本 + 检查清单）
- ✅ 度量驱动（自动周报）
- ✅ 开箱即用（< 30 分钟）

---

## 💰 商业模式

**Phase 1（当前）**：开源 + 品牌建设
- GitHub 开源仓库
- 文档和教程免费
- 建立社区和案例

**Phase 2（3-6 个月）**：增值服务
- 付费咨询（帮助定制）
- 企业版（多人协作、权限管理）
- SaaS 托管版（无需自建）

**Phase 3（6-12 个月）**：平台化
- Skill 市场（买卖可复用能力）
- Agent 市场（预训练的 agent）
- 数据集市场（行业数据）

---

## 🔬 验证计划

**Validation 阶段目标**：证明有人愿意使用

### Week 1-2：内部验证
- [ ] 自己用 org-v2 运行 2 周
- [ ] 至少完成 5 个真实任务
- [ ] 沉淀率保持 ≥ 30%
- [ ] 生成至少 5 个能力资产

### Week 3-4：外部验证
- [ ] 发布到 GitHub
- [ ] 撰写详细 README 和教程
- [ ] 在 Twitter/HN 分享
- [ ] 目标：10 个 star + 2 个外部使用者

### Month 2-3：深度验证
- [ ] 收集 3 个用户反馈
- [ ] 至少 1 个用户持续使用 > 1 个月
- [ ] 有人询问付费服务

### 停止标准
如果满足以下 2 条，停止：
- 2 个月无外部使用者
- 自己不再持续使用
- 无人询问付费可能性

## 📝 附录

### 技术栈
- 基础：Markdown + Shell scripts
- CI/CD：GitHub Actions
- 协作：Git + GitHub
- 未来：可能需要 Python/Node.js（度量可视化）

### 竞品
- org-blueprint（过于复杂）
- 传统项目管理工具（非 AI 原生）
- AI agent 框架（技术导向，缺少组织层）

### 风险
- 市场太小众（缓解：先自用验证价值）
- 开源难以变现（缓解：咨询 + 企业版）
- 维护负担（缓解：极简设计 + 社区）
