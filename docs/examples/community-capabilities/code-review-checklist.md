# 社区能力示例：代码审查检查清单

**贡献者**：org-v2 团队  
**场景**：Pull Request 代码审查  
**适用**：所有开发团队

---

## 🎯 用途

标准化代码审查流程，提升代码质量。

## ✅ 检查清单

### 1. 功能正确性

- [ ] 代码实现符合需求
- [ ] 边界情况已考虑
- [ ] 错误处理完整
- [ ] 测试覆盖充分

### 2. 代码质量

- [ ] 命名清晰易懂
- [ ] 函数职责单一
- [ ] 避免重复代码
- [ ] 复杂逻辑有注释

### 3. 测试

- [ ] 新功能有单元测试
- [ ] 边界情况有测试
- [ ] 测试名称描述清晰
- [ ] 所有测试通过

### 4. 文档

- [ ] README 已更新（如需要）
- [ ] API 文档已更新
- [ ] 复杂逻辑有注释
- [ ] 变更日志已更新

### 5. 安全性

- [ ] 无 SQL 注入风险
- [ ] 无 XSS 风险
- [ ] 敏感数据已加密
- [ ] 权限检查完整

### 6. 性能

- [ ] 无 N+1 查询
- [ ] 大数据集有分页
- [ ] 缓存使用合理
- [ ] 资源及时释放

---

## 📋 沉淀为能力

```bash
cat > capabilities/data/code-review-checklist.md << CAP
# Data: Code Review Checklist

**类型**：data
**创建**：YYYY-MM-DD

## 🎯 用途
Pull Request 代码审查标准检查清单

## ✅ 检查项（完整清单见上）

## 💡 使用方法
1. 创建 PR 时复制清单到描述
2. 审查者逐项检查
3. 所有项通过后合并
4. 定期回顾更新清单
CAP
```
