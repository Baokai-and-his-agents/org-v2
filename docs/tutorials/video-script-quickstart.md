## 📹 场景 4：理解能力概念（3:30-5:00）

**画面**：浏览 capabilities 目录

**旁白**：
```
核心概念：能力（Capability）。

能力有 4 种类型：

1. Skill - 工作方法（可重复的流程）
   比如：问题驱动开发、自动化质量检查

2. Data - 结构化事实（模板、清单）
   比如：产品 brief 模板、发布检查清单

3. Knowledge - 经验教训（案例、洞察）
   比如：为什么 Bash 跨平台会遇到问题

4. System - 系统设计（架构、模式）
   比如：健康度评分系统设计

每个能力都是一个 Markdown 文件，
包含：用途、输入、输出、步骤。

这就是知识沉淀的容器。
```

**画面切换**：
- 打开一个 skill 文件
- 显示结构
- 高亮关键部分

**关键信息**：
- 4 种能力类型
- Markdown 格式
- 标准化结构

## 📹 场景 5：完成第一个任务（5:00-7:30）

**画面**：创建任务文件

**旁白**：
```
现在我们完成第一个真实任务。

假设你要"优化团队周报流程"。

第一步：创建任务文件
ops/tasks/001-optimize-weekly-report.md

填写任务描述、输入、输出、验收标准。

第二步：执行任务
（这里是你实际工作的过程）
比如：设计新流程、编写脚本、测试...

第三步：关键来了 - 能力沉淀
任务完成后，问自己：
"这个经验能复用吗？"

如果能，创建能力文件：
capabilities/skill/weekly-report-automation.md

记录：为什么需要、怎么做、注意什么。

以后遇到类似问题，直接复用。
不用每次都重新思考。
```

**命令演示**：
```bash
cat > ops/tasks/001-optimize-weekly-report.md << 'TASK'
# 任务：优化周报流程

**目标**：自动化周报生成
...
TASK

# 执行任务...

cat > capabilities/skill/weekly-report-automation.md << 'CAP'
# Skill: Weekly Report Automation
...
CAP
```

**关键信息**：
- 任务 → 执行 → 沉淀
- 强制评估可复用性

## 📹 场景 6：搜索和复用（7:30-9:00）

**画面**：使用搜索工具

**旁白**：
```
积累几个能力后，怎么找到它们？

使用搜索工具：
bash ops/scripts/search-capability.sh "report"

系统会搜索所有能力，
找到相关的 skills、data、knowledge。

找到后，可以：
1. 直接使用（复制步骤）
2. 导出分享（跨团队复用）
3. 改进优化（持续迭代）

这就是复利效应：
第 1 个能力价值 = 1
第 10 个能力价值 = 10 + 之前的复用
第 100 个能力价值 = 指数级

传统方式每次从零开始，
org-v2 是站在巨人肩膀上。
```

**命令演示**：
```bash
bash ops/scripts/search-capability.sh "report"
bash ops/scripts/export-capability.sh -t skill -n weekly-report-automation
```

**关键信息**：
- 搜索工具
- 复利效应
- 指数增长
