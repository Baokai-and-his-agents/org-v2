# FAQ - Frequently Asked Questions

## 基础概念

### 什么是 org-v2？

org-v2 是一个 AI 原生组织框架，帮助个人和小团队通过强制能力沉淀机制实现 8.4 倍的学习效率提升。

它是一个基于 Git 的文件结构 + 工作流程，不是软件应用。

### 什么是"AI 原生组织"？

AI 原生组织是指从设计之初就为人类-AI 协作优化的组织结构，而不是在传统组织上加一层 AI 功能。

特征：
- AI 是协作者，不是工具
- 知识沉淀是核心流程
- 度量驱动改进
- 极简可执行

### 什么是"能力沉淀"？

能力沉淀是指将一次性工作转化为可复用能力的过程。

例如：
- 写一个脚本 → 沉淀为 skill（工作方法）
- 解决一个问题 → 沉淀为 knowledge（经验案例）
- 创建一个模板 → 沉淀为 data（结构化事实）

org-v2 的核心创新是**强制沉淀**：每个任务必须评估"能否复用"。

### org-v2 vs org-blueprint？

| 维度 | org-v2 | org-blueprint |

| **定位** | 极简可执行 | 完整治理框架 |
| **复杂度** | 3 层 | 5 层 |
| **上手时间** | 10 分钟 | 60 分钟 |
| **沉淀机制** | 强制（117%） | 可选（14%） |
| **沉淀效率** | **8.4 倍** | 基准 |
| **文档长度** | 50 行 README | 139 行 |
| **适合场景** | 立即开始使用 | 深入理解概念 |

**选择建议**：
- 想快速上手 → org-v2
- 想深入研究 → org-blueprint
- 两者不冲突，可以都看

### org-v2 vs AutoGPT/LangChain？

这些是不同层次的工具：

**AutoGPT/LangChain**：AI agent 开发框架（技术层）
- 用途：开发 AI 应用
- 输出：代码
- 用户：开发者

**org-v2**：AI 原生组织框架（组织层）
- 用途：运营 AI 协作组织
- 输出：能力资产
- 用户：运营者

可以结合使用：用 LangChain 开发 agent，用 org-v2 管理组织。

### org-v2 vs Notion/Asana？

**Notion/Asana**：传统协作工具 + AI 功能
- AI 是附加功能
- GUI 界面
- 订阅制
- 适合非技术团队

**org-v2**：AI 原生组织框架
- AI 是核心协作者
- Git/CLI 界面
- 免费开源
- 适合技术团队

选择取决于你的背景和需求。

### 我需要安装什么软件？

**必需**：
- Git
- 文本编辑器（VS Code/Vim/任意）

**可选**：
- GitHub CLI（`gh`）- 用于 PR/Issue 操作

### 我不会 Git，能用吗？

建议先学习 Git 基础（30 分钟）：
- `git clone`
- `git add`
- `git commit`
- `git push`

这些是现代开发的必备技能，值得学习。

### 能否 GUI 版本？

当前版本是纯 Git/CLI。

GUI 版本在考虑中，但不是优先级。原因：
- Git 本身有很多 GUI 工具
- CLI 更适合自动化
- 保持极简原则

## 技术细节

### 为什么基于 Git？

**优势**：
- 版本控制（所有改动可追溯）
- 分支管理（多任务并行）
- 协作友好（Pull Request）
- 免费可靠
- 无需额外服务器

**替代方案考虑过吗**？
考虑过数据库、云服务等，但 Git 更符合极简原则。

### 数据存在哪里？

**本地**：你的电脑（`.git` 目录）

**远程**：你选择的 Git 托管（GitHub/GitLab/自建）

你完全拥有数据，没有锁定风险。

### 如何备份？

Git 自动提供多重备份：
- 本地仓库（你的电脑）
- 远程仓库（GitHub 等）
- 任何 clone 的副本

推送到远程就是备份。

### 能否团队协作？

**可以**，但当前优化是 1-5 人。

协作方式：
- 每人 fork 或在自己分支工作
- Pull Request 合并
- 能力资产共享

大团队需要额外的权限管理（计划中）。

## 效果与验证

### "8.4 倍效率"是如何计算的？

**计算方法**：

org-v2 沉淀率：117%（7 能力 / 6 任务）
org-blueprint 沉淀率：14%（2 能力 / 14 任务）

比率：117% / 14% = 8.36 ≈ 8.4 倍

**这是真实的吗**？
- 来自实际运行数据
- 样本量较小（6 vs 14 任务）
- 需要更多验证（期待社区反馈）

### 数据是否樱桃采摘（cherry-picked）？

**不是**。

- 所有 6 个任务都计入
- 包括"失败"的 task 004（但也产生了能力）
- 完整记录在 Git 历史中
- 透明可验证

### 能否保证我也能达到 8.4 倍？

**不能保证**，因素包括：
- 任务类型
- 个人习惯
- 沉淀质量标准

但机制相同：强制沉淀 vs 可选沉淀。

我们预期你能达到 **3-5 倍**提升（保守估计）。

### 如何验证效果？

**自己测试**：
1. 记录当前沉淀率（过去 10 个任务产生多少能力）
2. 使用 org-v2 完成 10 个任务
3. 对比沉淀率变化

org-v2 内置度量，自动追踪。
