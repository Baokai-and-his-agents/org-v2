# 使用案例集

真实场景下 org-v2 的应用示例。

---

## 📚 案例列表

### 1. 个人开发者

**场景**：独立开发者管理多个项目

**挑战**：
- 项目切换频繁
- 经验难以复用
- 进度不可见

**使用 org-v2**：
- 每个项目一个 org-v2 实例
- 能力库在项目间共享
- NOW.md 追踪当前工作
- 周报自动生成

**成效**：
- 切换成本降低 60%
- 能力复用率 50%+
- 进度清晰可见

[查看详情](./personal-developer.md)

### 2. 小型技术团队（2-5 人）

**场景**：初创公司技术团队

**挑战**：
- 知识散落各处
- 新人上手慢
- 质量不稳定

**使用 org-v2**：
- 共享能力库
- 强制沉淀流程
- CI 质量门禁
- 定期健康度检查

**成效**：
- 新人上手时间减半
- 代码质量提升 40%
- 知识留存率 70%+

[查看详情](./small-tech-team.md)

---

### 3. AI 研究项目

**场景**：AI 实验与论文项目

**挑战**：
- 实验结果难追踪
- 代码难复现
- 知识分散

**使用 org-v2**：
- 实验作为任务
- 结果沉淀为 knowledge
- 代码沉淀为 skill
- 数据集沉淀为 data

**成效**：
- 实验可复现
- 知识结构化
- 写作效率提升

[查看详情](./ai-research.md)

---

### 4. 开源项目维护

**场景**：维护开源项目

**挑战**：
- 贡献者流动
- 知识断层
- 质量参差

**使用 org-v2**：
- 贡献模板（data）
- Review checklist（data）
- 架构决策（knowledge）
- CI/CD 流程（skill）

**成效**：
- 贡献质量提升
- 维护者减负
- 社区活跃度提高

[查看详情](./open-source-project.md)

---

### 5. 产品开发

**场景**：从 0 到 1 构建产品

**挑战**：
- 需求频繁变化
- 技术债累积
- 进度不可控

**使用 org-v2**：
- 需求变更记录（knowledge）
- 技术决策沉淀（knowledge）
- 模板快速复用（data）
- 进度可视化（NOW.md）

**成效**：
- 迭代速度提升 30%
- 技术债可控
- 决策有据可查
