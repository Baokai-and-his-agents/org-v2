---

### 优先级评估

我们用这个框架评估反馈优先级：

```
优先级分数 = 影响人数 × 严重程度

影响人数：
- 高（3）：大多数用户
- 中（2）：部分用户
- 低（1）：个别用户

严重程度：
- 阻塞（3）：无法使用
- 重要（2）：体验受影响
- 轻微（1）：小问题

示例：
- 9 分（3×3）：P0，立即修复
- 4 分（2×2）：P1，本周修复
- 1 分（1×1）：P2，计划修复
```

## 💡 好反馈示例

### ✅ 好的反馈

**示例 1：具体问题**
```
在 macOS 上运行 duty-check.sh 时遇到错误：
"wc: illegal option -- l"

环境：macOS 13.4, zsh
重现步骤：bash ops/scripts/duty-check.sh
期望：显示统计信息
实际：报错退出
```

**为什么好**：
- 环境清晰
- 可复现
- 期望明确

---

**示例 2：改进建议**
```
建议：添加能力搜索功能

场景：我有 50+ 能力，想找"错误处理"相关的
当前：手动浏览 capabilities/ 目录
建议：bash ops/scripts/search-capability.sh "error"
预期：列出所有包含 "error" 的能力

优先级：中（不阻塞，但会提升体验）
```

**为什么好**：
- 场景清晰
- 具体建议
- 自评优先级

---

### ❌ 不够好的反馈

**示例 1：太模糊**
```
这个东西不太好用
```

**改进建议**：
- 哪里不好用？
- 期望是什么？
- 遇到什么问题？

---

**示例 2：缺乏上下文**
```
脚本报错了
```

**改进建议**：
- 哪个脚本？
- 什么错误？
- 什么环境？
