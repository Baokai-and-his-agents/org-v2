## 🎯 目标

开发一个自动化脚本，检查能力资产的质量和一致性，确保 capabilities/ 目录的健康度。

---

## 📋 背景

随着能力资产增长，需要确保：
- INDEX.md 与实际文件同步
- 每个能力文档格式完整
- 无过期或无效链接
- 质量标准一致

---

## 📊 检查项

### 1. 数量同步检查
- [ ] 统计实际文件数（skill/data/knowledge）
- [ ] 对比 INDEX.md 声明的数量
- [ ] 如不一致，报告差异

### 2. 文件格式检查
- [ ] 每个能力文档有必需字段（类型、创建日期、用途）
- [ ] 有示例或使用记录
- [ ] frontmatter 格式正确

### 3. 链接有效性
- [ ] 检查内部链接（[[name]]）
- [ ] 确认被引用的能力存在
- [ ] 报告死链

### 4. 过期检测
- [ ] 超过 3 个月未更新 → 标记为待审查
- [ ] 复用次数为 0 → 标记为未使用

---
