## 🎯 目标

开发一个自动同步脚本，确保 capabilities/INDEX.md 中的统计数据与实际文件数量始终一致，减少手动维护工作。

---

## 📋 背景

当前问题：
- 每次新增能力都要手动更新 INDEX.md
- 统计数据容易不同步
- check-capabilities.sh 只检查，不自动修复

需要：
- 自动统计实际文件数
- 自动更新 INDEX.md
- 可以手动触发或 PR 时自动运行

---

## 📊 任务范围

### In Scope
- 自动统计 skills/data/knowledge 文件数
- 自动更新 INDEX.md 统计部分
- 生成更新日志
- 验证同步成功

### Out of Scope
- 自动生成能力清单（保持手动）
- 修改能力文档内容
- 复杂的 diff 算法

---

## 🔄 执行计划

### 1. 实现统计逻辑
- [ ] 统计每个类型的文件数
- [ ] 排除 README.md
- [ ] 生成新的统计数据

### 2. 更新 INDEX.md
- [ ] 读取当前 INDEX.md
- [ ] 替换统计部分
- [ ] 保持其他内容不变

### 3. 验证与报告
- [ ] 对比更新前后
- [ ] 生成更新日志
- [ ] 验证格式正确

### 4. 集成到工作流
- [ ] 作为独立脚本可用
- [ ] 可被 CI 调用
- [ ] 提供 dry-run 模式
