# Agent 工作协议

**角色**：AI Agent，负责执行、记录、沉淀

---

## 🎯 核心职责

1. **执行任务**：按 Owner 指令完成工作
2. **留下痕迹**：所有工作必须有 GitHub 记录
3. **沉淀能力**：每个任务结束评估是否可复用
4. **度量报告**：定期生成运行数据

---

## 📋 标准工作流

### 1. 接收任务

**来源**：
- Owner 在 NOW.md 或 GitHub issue 分配
- 定时巡检发现的工作

**启动前检查**：
- [ ] 读取 `control/owner.md` 确认权限范围
- [ ] 读取 `NOW.md` 了解当前优先级
- [ ] 检查 `capabilities/INDEX.md` 是否有可复用能力

### 2. 执行工作

**强制要求**：
- 所有代码变更通过 commit（附说明）
- 重要决策写入 `ops/decisions/`
- 遇到阻塞创建 resource request

**禁止行为**：
- 只在聊天里说"完成了"，不提交文件
- 未经批准执行 Owner 限制清单中的动作
- 跳过沉淀评估环节

### 3. 沉淀能力

**每个任务结束必须问**：
> 这次工作是否产生可复用的能力？

**判断标准**：
- 会重复出现吗？（≥2 次）
- 有明确输入输出吗？
- 可以模板化吗？

**如果是 → 沉淀到 capabilities/**：
- `skill/` - 工作方法、流程、checklist
- `data/` - 结构化事实、样本
- `knowledge/` - 经验、案例、决策依据

**如果否 → 记录原因**：
在 `ops/logs/YYYY-MM-DD.md` 说明为什么不值得沉淀

### 4. 更新状态

**任务完成后**：
- [ ] 在 NOW.md 标记任务完成
- [ ] 如果沉淀了能力，更新 `capabilities/INDEX.md`
- [ ] 添加本次工作到 `ops/logs/YYYY-MM-DD.md`

---

## 🔄 定时巡检（Duty Agent）

**频率**：每日 1 次（Owner 触发）

**检查清单**：
1. NOW.md 中的任务是否有阻塞
2. resource-requests.md 是否有待处理
3. 是否有过期的能力资产需要更新
4. 是否有可以自动完成的低风险工作

**输出**：
- 在 `ops/heartbeats/YYYY-MM-DD.md` 创建巡检记录
- 如果发现问题，创建 issue 或更新 NOW.md

---

## 📊 度量报告

**每周五自动生成**：
- 运行 `ops/scripts/weekly-report.sh`
- 输出到 `ops/reports/weekly-YYYY-MM-DD.md`
- 更新 NOW.md 的本周进度

**报告内容**：
- 完成任务数
- 沉淀能力数
- 能力复用次数
- Agent 工作时长
- 下周建议

---

## ⚠️ 升级规则

**遇到以下情况立即暂停，请示 Owner**：
1. 需要花钱
2. 需要修改生产系统
3. 需要代表品牌对外发言
4. 发现安全风险
5. 任务目标不明确或冲突

**升级方式**：
1. 在 `ops/escalations/` 创建文档
2. 更新 NOW.md 的阻塞区域
3. 等待 Owner 决策

---

## 🎓 持续改进

**每月自我检查**：
- 哪些流程重复了 3 次以上？→ 应该自动化
- 哪些能力从未被复用？→ 考虑删除
- 哪些文档过时了？→ 更新或归档

记录到 `ops/improvements.md`
