# 示例：个人博客管理系统

使用 org-v2 管理个人博客的完整示例。

---

## 📌 场景描述

**角色**：独立开发者 / 内容创作者  
**目标**：高效管理博客内容，持续积累写作和运营能力  
**挑战**：写作、发布、维护工作重复，没有系统沉淀

---

## 🎯 使用 org-v2 的价值

### 传统方式（无系统）

1. 每次写文章都要想"写什么"
2. 发布流程每次都要查文档
3. SEO 检查靠记忆，容易遗漏
4. 没有复盘，不知道什么内容有效

### 使用 org-v2

1. ✅ 文章创意沉淀为 idea-bank（data）
2. ✅ 发布流程固化为 publish-checklist（skill）
3. ✅ SEO 检查自动化（skill）
4. ✅ 数据分析沉淀为 insights（knowledge）

**结果**：写作效率提升 3 倍，质量更稳定

---

## 📂 目录结构

```
personal-blog/
├── control/
│   └── owner.md           # 博客管理者控制面板
├── ops/
│   ├── tasks/             # 任务记录（写文章、优化SEO等）
│   └── scripts/           # 自动化脚本（发布、分析）
└── capabilities/
    ├── skill/             # 工作方法（写作流程、SEO检查）
    ├── data/              # 数据（文章模板、idea库）
    └── knowledge/         # 经验（什么内容受欢迎）
```

---

## 🔄 典型工作流

### Week 1: 写第一篇文章

**任务**：写一篇技术教程

**过程**：
1. 在 NOW.md 添加任务："写 React Hooks 教程"
2. 使用通用文章模板（后来沉淀为 data/article-template.md）
3. 写作 → 校对 → 发布
4. 发布后检查 SEO（手动，后来自动化）

**沉淀**：
- data/article-template.md（文章结构模板）
- skill/seo-checklist.md（SEO 检查清单）

---

### Week 2: 写第二篇文章

**任务**：写一篇产品评测

**过程**：
1. 直接复用 article-template.md（节省 20 分钟）
2. 使用 seo-checklist.md 自检（避免遗漏）
3. 发布流程更顺畅

**沉淀**：
- skill/publish-workflow.md（发布完整流程）

---

### Week 4: 数据复盘

**任务**：分析前 4 篇文章数据

**过程**：
1. 收集阅读量、留言、分享数
2. 分析什么类型内容受欢迎
3. 提炼规律

**沉淀**：
- knowledge/content-insights.md（内容洞察）
  - 教程类阅读量高
  - 实战类分享多
  - 理论类完读率低

---

### Week 8: 持续加速

**效果**：
- 写作时间：3 小时 → 1.5 小时（-50%）
- SEO 遗漏：30% → 0%（-100%）
- 选题方向：凭感觉 → 数据驱动

**原因**：
- 复用模板和流程
- 避免重复决策
- 基于数据而非直觉

## 📚 沉淀的能力示例

### Skill: SEO Checklist

```markdown
# Skill: 文章 SEO 检查清单

## 📋 发布前检查

- [ ] 标题包含关键词（前 60 字符）
- [ ] 描述（meta description）吸引人（120-160 字符）
- [ ] 首段包含核心关键词
- [ ] 至少 3 个 H2 标题
- [ ] 图片有 alt 文本
- [ ] 内部链接（2-3 个）
- [ ] 外部权威链接（1-2 个）
- [ ] URL 简洁可读
- [ ] 移动端预览

## 📊 发布后检查

- [ ] Google Search Console 提交
- [ ] 社交媒体分享预览正常
- [ ] 加载速度 <3 秒

**复用次数**：每篇文章 1 次
```

### Data: Article Template

```markdown
# Data: 技术文章模板

## 标题（60 字符以内，包含关键词）

## 简介（120-160 字符）

核心价值 + 读者收获

## 正文

### 问题背景

为什么需要这个？

### 解决方案

如何做？（步骤 + 代码）

### 实际应用

真实案例

### 总结

关键要点 + 下一步

## 相关阅读

- 内部链接 1
- 内部链接 2

## 元数据

- 关键词：[主关键词, 次关键词]
- 分类：[技术/产品/运营]
- 标签：[tag1, tag2, tag3]
```

---

### Knowledge: Content Insights

```markdown
# Knowledge: 内容洞察（2026-06）

## 数据来源

- 样本：10 篇文章
- 时间：2026-01 - 2026-06

## 📊 发现

### 1. 教程类表现最好

**数据**：
- 平均阅读量：2500（vs 整体 1200）
- 完读率：65%（vs 整体 45%）
- 分享率：12%（vs 整体 6%）

**原因**：
- 实用性强
- 可复现
- 有代码示例

**行动**：优先创作教程内容

### 2. 标题长度影响点击

**数据**：
- 40-50 字符：点击率 8%
- 50-60 字符：点击率 12%
- 60+ 字符：点击率 6%

**行动**：标题控制在 50-60 字符

### 3. 发布时间影响阅读

**数据**：
- 周二 8-10am：最佳（阅读量 +40%）
- 周末：较差（阅读量 -30%）

**行动**：周二早上发布
```

## 💬 常见问题

**Q: 我的博客很简单，需要这么复杂吗？**  
A: org-v2 不复杂，只是系统化。一开始可能看起来"多余"，但 5-10 篇文章后，你会感受到复利。

**Q: 我不是技术博客，适用吗？**  
A: 适用。任何需要持续创作的场景都能受益。调整模板和检查清单即可。

**Q: 沉淀能力会不会占用太多时间？**  
A: 每次沉淀 5-10 分钟，但复用时节省 30-60 分钟。投入产出比 1:5+。
