## 🎯 用途

记录工具链设计的关键决策和经验教训。

## 💡 核心洞察

### 1. 完整闭环 > 单点工具

**错误**：只有导出，没有导入  
**正确**：导出 → 导入 → 搜索（完整循环）

**收益**：
- 能力可流转
- 跨项目共享
- 生态建立基础

### 2. 跨平台兼容性优先

**挑战**：macOS 和 Linux 的 `date` 命令不同

**方案**：
```bash
if [[ "$OSTYPE" == "darwin"* ]]; then
    # macOS
    date -v-7d
else
    # Linux
    date -d "-7 days"
fi
```

**经验**：早期考虑，避免后期返工

### 3. 优雅降级

**示例**：导入工具支持 yq + fallback 解析器

**原因**：
- yq 不是所有人都有
- 100% bash fallback 保证可用性

### 4. 自动化 > 手动

**测试**：
- 手动质量检查：容易忘记
- 自动化 CI 门禁：强制执行

**收益**：持续质量保障

## 🎓 设计原则

1. 完整性 > 功能多
2. 简单 > 复杂
3. 跨平台 > 单平台
4. 自动化 > 手动
5. 渐进式 > 一次性
