# System: Health Scoring System

**类型**：system  
**创建**：2026-06-13  
**更新**：2026-06-13  
**复用次数**：10+

## 🎯 用途

8 维度健康度评分系统的设计与实现。

## 📊 8 个维度

1. **结构清晰度** (0-10): 目录合理性、命名一致性
2. **可执行性** (0-10): 脚本可运行、文档可操作
3. **文档完整度** (0-10): 核心文档齐全、内容详尽
4. **自动化程度** (0-10): CI/CD 配置、自动化脚本数量
5. **能力沉淀率** (0-10): 沉淀数 / 任务数，目标 ≥ 70%
6. **度量可见性** (0-10): 进度可追踪、报告可生成
7. **质量保障** (0-10): 测试覆盖、质量门禁
8. **外部验证** (0-10): 外部用户反馈、社区活跃度

## 🔄 计算方法

总分 = (S1 + S2 + ... + S8) / 8

## 💡 评分标准

- 9.0-10.0: 优秀 (Excellent)
- 8.0-8.9: 良好 (Good)
- 7.0-7.9: 合格 (Fair)
- <7.0: 需改进 (Needs Improvement)

_设计于 Phase 3，验证于 Phase 5_
